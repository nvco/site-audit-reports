# Site Audit Reports

Static repo for sharing site audit reports generated by [site-audit-cli](https://github.com/nvco/site-audit-cli). Each client gets a folder under `reports/`. Reports are shared via direct links — there is no top-level navigation.

## Structure

```
reports/
  style.css           # shared stylesheet, referenced by all index.html files
  {client}/
    index.html        # landing page for this client's reports
    {slug}-{YYYYMMDD}-{HHMMSS}.html
    {slug}-{YYYYMMDD}-{HHMMSS}.pdf
    {slug}-{YYYYMMDD}-{HHMMSS}.md
    {slug}-{YYYYMMDD}-{HHMMSS}.json
```

## index.html format

Each client folder has an `index.html` that lists all audits with links to each format. To recreate or update it:

1. Read the `.md` files in the folder to get: `Target:`, `Run date:`, `Pages audited:`
2. Group files by their `{slug}-{YYYYMMDD}-{HHMMSS}` base name — each base name is one audit
3. Extract the time from the filename (`HHMMSS`) and format as 12-hour (e.g. `131319` → `1:13 PM`)
4. Use the domain from `Target:` as the `<h2>` heading (strip `https://` and trailing paths)
5. The stylesheet is always `../style.css`

### Template

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Site Audit Reports — {Client Name}</title>
  <link rel="stylesheet" href="../style.css">
</head>
<body>
  <div class="wrapper">
    <header>
      <h1>Site Audit Reports: {Client Name}</h1>
      <p>Automated audits covering accessibility, privacy, cookies, security headers, SSL/TLS, and broken links. Conducted {Month D, YYYY}.</p>
    </header>

    <!-- repeat for each audit, ordered by run time ascending -->
    <div class="report">
      <div class="report-header">
        <h2>{domain from Target}</h2>
        <div class="meta">{N} pages &middot; {Month D, YYYY} at {H:MM AM/PM}</div>
      </div>
      <div class="files">
        <a class="file" href="{base}.html">
          <span class="file-label">HTML</span>
          <span class="file-desc">Interactive report: open and browse in your browser</span>
        </a>
        <a class="file" href="{base}.pdf">
          <span class="file-label">PDF</span>
          <span class="file-desc">Printable version: good for sharing or review</span>
        </a>
        <a class="file" href="{base}.md">
          <span class="file-label">MD</span>
          <span class="file-desc">Markdown format: best for AI tools, developers, and plain text readers</span>
        </a>
        <a class="file" href="{base}.json">
          <span class="file-label">JSON</span>
          <span class="file-desc">Structured data file for developers and automated tools</span>
        </a>
      </div>
    </div>

  </div>
</body>
</html>
```

### File descriptions (keep these consistent)

| Format | Label | Description |
|--------|-------|-------------|
| .html  | HTML  | Interactive report: open and browse in your browser |
| .pdf   | PDF   | Printable version: good for sharing or review |
| .md    | MD    | Markdown format: best for AI tools, developers, and plain text readers |
| .json  | JSON  | Structured data file for developers and automated tools |
